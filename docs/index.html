Dynamic Context Evolution for Scalable Synthetic Data Generation

Ryan Lingo1, Rajeev Chhajer1
1Honda Research Institute, USA, Inc.
arXiv Code Data

The Problem

When you need thousands of synthetic training examples from an LLM, you prompt it in batches. But each API call starts with a blank context window. The model has no memory of what it already generated, so it keeps returning to the same high-probability outputs.

We call this cross-batch mode collapse. Ask for 1,000 ideas and you'll get the same 50 concepts rephrased over and over. By batch 200, over a third of the output is redundant.

How DCE Works

DCE gives the model a memory of everything it has already generated, and a prompt that adapts based on what's missing. Three mechanisms run each batch:

1. Verbalized tail sampling asks the model to rate how obvious each idea is. Predictable ideas (like "smart water bottle") are filtered out. Only surprising, low-probability ideas are kept.

2. Semantic memory stores every accepted idea as an embedding in a vector database. Before a new idea is accepted, the system checks whether anything semantically similar already exists. This catches duplicates even when they use completely different words.

3. Adaptive prompt evolution rewrites the generation prompt every batch based on what the memory contains. It tracks which categories have too many ideas and which have too few, then steers the model toward the gaps.

Results

DCE achieves 0.0% collapse versus 5.6% for naive prompting, tested across three domains, two model families, and multiple random seeds. It consistently produces 17-18 distinct concept clusters where naive prompting produces anywhere from 2 to 17.

No single component is sufficient on its own. Deduplication alone doesn't stop the model from generating the same ideas. Prompt evolution alone doesn't catch near-duplicates. The three mechanisms have to work together.

The cost is roughly $0.50 per 1,000 candidates using standard API calls. No fine-tuning or custom architectures required.
